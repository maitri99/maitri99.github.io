Chocolate Bar Ratings

April 14, 2021| Category: Data Preprocessing and EDA

In order to explore the data and visualize it, some changes might need to be made to the data along the way. This is often referred to as data preprocessing. Though EDA is technically different from data preprocessing, EDA often exposes problems with the data that need to be fixed in order to continue exploring. Because of this tight overlap, we'll clean the data as per the needs to help understand the data.
This project uses Pandas knowledge to explore a dataset about chocolate. I will explore each column in our dataset individually as well as take the results of our preprocessed data and search for patterns across columns and rows. This project uses a chocolate bar ratings dataset from Kaggle. The dataset is from the Flavors of Cacao data.

There are 1,795 rows and 9 columns in the dataset. Acquire the data on your machine and run a quick fact check. You will see that the data seems to be sorted by the 'Company' column. There is definitely some missing data in the 'Bean Type' column. One of the more frustrating aspects of this dataset is the poor format of the column names. Typing 'Specific Bean Origin\nor Bar Name' in order to access the column is painful. So our first step would be to modify column names! Then, you will notice that the columns are also in an odd order. Information about the company is spread across the columns, and so is the information about the cacao bean. Hence, we should order the columns a little more meaningfully according to the Company Information, Chocolate Bar Information, and Review Information. We can reorder the columns by specifically selecting the columns in order and reassigning them to the dataframe variable

Examining Each Column

The 'Company' column is the first in the list, so let's check it out first. We can tell that the column contains string values. See if we have any missing values and the amount of distinct values. Upon checking, we will notice that this is some interesting data. Looking at it raises many questions. For instance:
Should company names like 'Vintage Plantations' and 'Vintage Plantations (Tulicorp)' be changed to the same name?
Is 'Cacao de Origin' a misspelling of 'Cacao de Origen'?
Is 'Shattel' a misspelling of 'Shattell'?
These are some things we see and questions we ask when we encounter a new dataset. Rarely is the data in perfect condition. Often we spend a significant amount of time researching topics related to the data in order to make a call about modifying aspects of the data.
In this particular project, we could cross reference the names in the dataset with the names in the master list provided, we find a master list of all of the chocolate makers in the world. Unfortunately, we don't have a master list of chocolate makers. Instead, we will have to manually inspect the data and research when things don't look right. For example, fixing the misspellings.

Now, let's move into data about the cocoa going into the chocolate bar itself. The first piece of data is the 'Bean Type' which is defined as "The variety (breed) of bean used, if provided". This prevails that there will be some missing data.
Upon checking, we will see that only one row of data is missing 'Bean Type'. Should we leave it as is, remove the entire row, or fill in the data with some value?
Leaving undefined values lying around in our data can be problematic. Missing values are not counted and can be tricky to program around. Removing the entire row actually isn't a bad option in this case. Since it is only one row out of over 1,700, it likely won't have too much effect on any analysis that we do. In this case, we will replace the missing value with 'Unknown'.
Furthermore, there are non-breaking space characters, 0xa0 in the 'Bean Type' column. We will replace these values with the word 'Unknown'.

After finding and repairing the bad data, visualize the dataset to see any patterns. For example, create a visualization that shows the number of reviews that were created each year.

Upon taking a look at the ratings, we will see that ratings are floating point values and that they range from 1.0 to 5.0. But are they really continuous? Interestingly enough, the values don't seem to be continuous, but instead seem to be divided into quarters. Meaning, downstream we might be able to use a regression or categorical model in order to predict these values.

We've now explored every column in our dataset. We have filled in missing values and repaired obviously bad data. We will move forward to core EDA.
